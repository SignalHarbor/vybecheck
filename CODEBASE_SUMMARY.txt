--------------------------------------------------------------------------------
1. PURPOSE & TARGET USERS
--------------------------------------------------------------------------------

VybeCheck is an interactive quiz platform for Twitter Spaces that:
- Transforms live Twitter Spaces conversations into rapid-fire true/false quizzes
  (5–10 second answer windows)
- Matches participants based on RESPONSE AGREEMENT (not correctness) —
  participants with similar answers get higher compatibility scores
- Supports long-running sessions (2–3 months) for async participation after the
  Space ends
- Monetizes through a credit-based ("Vybes") system for premium features

Target Users:
- Twitter Space hosts wanting to engage audiences interactively
- Participants discovering vibe compatibility with others
- Premium users wanting detailed match insights

2. TECH STACK
--------------------------------------------------------------------------------

Frontend:
  - React 19 + Vite (SPA)
  - TypeScript (ES2022, strict mode)
  - Zustand (state management)
  - Tailwind CSS 4 + PostCSS
  - Lucide Icons
  - PostHog (analytics)

Backend:
  - Node.js (v20+) + Express.js (REST API)
  - WebSockets (ws library) — real-time quiz, responses, matches
  - TypeScript (strict mode)
  - SQLite (better-sqlite3) — persistent storage with write-through caching
  - Stripe (payment processing)
  - OpenAI API (transcription + question generation via GPT-4o + Whisper)
  - JWT + Twitter OAuth 2.0 with PKCE (authentication)
  - Pino (logging)

Deployment:
  - Backend:  Fly.io (WebSocket + API server, region: iad, 256 MB, 1 shared CPU)
  - Frontend: Cloudflare Pages (static SPA)
  - Database: SQLite mounted on Fly.io volume at /app/data

3. MAJOR FEATURES IMPLEMENTED
--------------------------------------------------------------------------------

A. Authentication
   - Twitter OAuth 2.0 with PKCE flow
   - JWT tokens (30-day expiration)
   - User profile persistence (Twitter ID, username, display name, avatar)
   - src/server/services/AuthService.ts — token exchange & validation
   - src/frontend/store/authStore.ts — client auth state

B. Quiz Management
   - Session Creation: host creates unique 6-character session IDs
   - Dynamic Question Addition: owners add questions (exactly 2 options each)
   - Real-time Responses: participants submit answers within timer window
   - Session Lifecycle:
       live → active (quiz ongoing) → expired (closed after 3 months)
   - src/server/models/QuizSession.ts — core session logic
   - src/server/models/Question.ts, Response.ts — data models

C. Matching Engine
   - Agreement-based: calculates match % between participants based on
     identical responses
   - Tiered Results (3 tiers, different visibility):
       PREVIEW  — 2 matches (free)
       TOP3     — Top 3 matches (2 Vybes)
       ALL      — All matches sorted by compatibility (5 Vybes)
   - Caching: 10-minute TTL, invalidated on quiz state changes
   - src/server/services/MatchingService.ts

D. Monetization System (Credit-Based "Vybes")
   - Initial Allocation: 10 Vybes per new user
   - Purchase Packs:
       Starter:  20 Vybes for $5
       Pro:      50 Vybes for $10  (popular)
       Ultimate: 120 Vybes for $20
   - Unlock Costs:
       MATCH_TOP3:          2 Vybes
       MATCH_ALL:           5 Vybes
       QUESTION_LIMIT_10:   3 Vybes (extends from 3 → 10 questions per session)
   - Idempotent purchases — no double-charging on refresh/retry
   - src/server/services/BillingService.ts — core billing logic
   - src/server/models/VybeLedger.ts — append-only transaction ledger
   - src/server/models/ParticipantUnlock.ts — feature unlock tracking
   - src/server/models/QuotaManager.ts — question limit enforcement

E. Payment Processing
   - Stripe Integration: checkout sessions, webhook handling
   - Automatic credit issuance on successful payment via Stripe webhook
   - src/server/services/StripeService.ts
   - src/server/routes/payment.ts

F. AI Features (Feature-Flagged)
   - Audio Transcription: upload MP3 → OpenAI Whisper API
   - Question Generation: GPT-4o generates quiz questions from transcripts
   - Validates MP3 format (ID3v2 or MPEG sync word)
   - src/server/services/QuestionGeneratorService.ts
   - src/server/services/TranscriptionService.ts
   - src/server/routes/ai.ts

G. WebSocket Real-time Communication
   - Session Messages:   create, join, reconnect, leave, start, terminate,
                         release-results
   - Question Messages:  add questions, unlock question limit
   - Response Messages:  submit answers
   - Match Messages:     request matches by tier
   - Credits Messages:   check balance, view history
   - Ping/Pong:          connection health checks
   - src/server/services/WebSocketHandler.ts (1000+ lines, core event handler)

H. Persistence & Database
   - SQLite Schema:
       Users               (Twitter profile)
       Sessions            (quiz metadata)
       Participants        (per-session)
       Questions           (per-session)
       Responses           (per-session)
       VybeLedger          (transactions)
       ParticipantUnlocks  (feature access)
       StripeCheckoutSessions (idempotency tracking)
   - Write-through caching: in-memory structs sync to DB
   - src/server/db/database.ts — schema init
   - src/server/db/repositories/ — DAO layer

I. Feature Flags
   - src/shared/features.ts — flag definitions
   - Flags: enablePayments, enableAIGeneration
   - Admin users (ADMIN_USERS array) bypass all flags
   - Dev mode auto-enables all flags

6. SERVER ROUTES & API ENDPOINTS
--------------------------------------------------------------------------------

  POST  /api/auth/twitter/token      Exchange OAuth code → JWT
  GET   /api/auth/me                 Validate token & return user
  POST  /api/checkout                Create Stripe checkout session
  GET   /api/checkout/verify         Verify session & award credits
  GET   /api/packs                   List available Vybe packs
  POST  /api/webhooks/stripe         Stripe webhook (raw body)
  GET   /api/vybes/balance           Get user's Vybe balance
  GET   /api/vybes/history           Transaction history
  POST  /api/vybes/issue             Admin: issue Vybes directly
  POST  /api/ai/generate-questions   Upload MP3 → transcribe + generate questions

WebSocket Client → Server message types:
  session:create, session:join, session:reconnect, session:leave,
  session:start, session:terminate, session:release-results,
  question:add, question:unlock-limit,
  response:submit,
  matches:get (with tier param),
  credits:balance, credits:history,
  ping

WebSocket Server → Client message types:
  Session events, quiz state updates, participant joins/leaves,
  question/response confirmations, match results (cost + balance),
  credit balance & transaction history, errors & notifications

12. ARCHITECTURAL PATTERNS
--------------------------------------------------------------------------------

  1. Vertical Slicing        Features built in testable slices
  2. Service-Oriented        Separate services per business domain
  3. Repository Pattern      DAO layer abstracts SQLite from business logic
  4. Write-Through Caching   In-memory models sync to DB automatically
  5. Idempotent Operations   Purchases tracked to prevent double-charging
  6. Tier Hierarchy          MATCH_ALL ⊇ MATCH_TOP3 ⊇ MATCH_PREVIEW
  7. Resource-Based Access   Unlocks tied to resourceId (e.g. session:abc123)
  8. WebSocket Pub/Sub       Broadcast to all participants on state changes
